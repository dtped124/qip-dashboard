# QIP 持續性監測指標 — 管制圖類型選型分析

> **作者立場**：以管制圖（SPC）專家角度，逐一分析每個指標的數據本質，判定最適管制圖類型。  
> **核心論點**：不是所有指標都適合用 I Chart（個別值管制圖）。管制圖的選擇取決於數據本質——是連續型量測、比率型（二項分布）、還是計數型（Poisson 分布），以及分母（觀察機會）是否逐月變動。

---

## 1. 管制圖選型決策框架

### 1.1 管制圖類型總覽

| 管制圖 | 適用數據 | 分布假設 | 典型場景 |
|--------|---------|---------|---------|
| **I-MR Chart**（個別值-移動全距） | 連續型數據，每期僅一個觀測值 | 近似常態 | 等候時間、住院天數、病床使用率 |
| **P Chart**（比例管制圖） | 二項比例型（pass/fail），分母可變 | 二項分布 | 手部衛生遵從率、手術部位感染率 |
| **NP Chart**（不良數管制圖） | 二項計數型（pass/fail），分母固定 | 二項分布 | 固定樣本中的不合格數 |
| **U Chart**（單位缺點率管制圖） | 計數率型（事件/暴露量），分母可變 | Poisson 分布 | 感染密度（件/千人日）、跌倒密度 |
| **C Chart**（缺點數管制圖） | 計數型，分母固定 | Poisson 分布 | 固定觀察期的事件計次 |
| **EWMA**（指數加權移動平均） | 需偵測小幅偏移的任何類型 | — | 高敏感度監測 |
| **CUSUM**（累積和管制圖） | 需偵測持續性微幅偏移 | — | 死亡率長期趨勢偵測 |

### 1.2 選型決策樹

```
你的數據是什麼類型？
│
├─ 連續型（量測值）
│   ├─ 每期一個數據點 → I-MR Chart
│   └─ 每期有子群組（多個觀測） → X̄-R 或 X̄-S Chart
│
└─ 離散型（計數/分類）
    │
    ├─ 二項型（每個個體 pass/fail）
    │   ├─ 分母（樣本數）每月變動 → P Chart
    │   └─ 分母每月固定 → NP Chart
    │
    └─ Poisson 計數型（事件次數/暴露量）
        ├─ 分母（暴露量如人日）每月變動 → U Chart
        └─ 分母每月固定 → C Chart
```

### 1.3 醫療品質指標的特殊考量

在醫療品質監測情境中，有幾個重要的實務考量會影響管制圖選型：

**考量一：分母變動問題**  
醫院每月的住院人次、手術件數、急診人次等都會波動。這意味著大多數比率型或密度型指標的分母是變動的，應使用 P Chart 或 U Chart 而非 NP Chart 或 C Chart。

**考量二：「已計算好的比率」問題**  
QIP 報表中匯入的數據通常是**已經計算好的比率**（如「1.2%」），而非原始的分子與分母。如果無法取得原始分子/分母數據，即使理論上應該用 P Chart 或 U Chart，實務上也只能退而求其次使用 I-MR Chart。

**考量三：稀有事件問題**  
某些指標的事件極為稀少（如每月 0-2 件），此時 P Chart 或 U Chart 的管制界限計算可能不穩定。Dr. Wheeler 建議，當 p̄ × n < 5 時，P Chart 的常態近似不成立，應改用 I-MR Chart 或精確二項分布界限。

**考量四：I-MR Chart 的穩健性**  
多篇文獻指出，I-MR Chart 對多種數據分布具有良好的穩健性（robustness）。SPC for Excel 的 Dr. McNeese 明確表示：「如果你不確定應該用哪種圖，用 I-MR Chart 通常不會出錯。」但這不代表 I-MR 永遠是最佳選擇——當數據確實符合 P Chart 或 U Chart 的假設時，使用正確的圖型能提供更精確的管制界限。

## 2. 逐一指標分析

以下針對每個 QIP 指標，從五個維度進行分析：

1. **數據本質**：連續/比率/計數
2. **分子/分母結構**：是否可取得原始值
3. **分布特性**：二項/Poisson/近似常態
4. **分母穩定性**：每月觀察機會是否變動
5. **稀有程度**：事件是否極為稀少

---

### 2.1 整體性照護

#### 指標：住院死亡率

| 維度 | 分析 |
|------|------|
| **數據本質** | 比率型（死亡人數 / 住院人次 × 1000） |
| **分子** | 住院期間死亡人數（計數，離散） |
| **分母** | 住院人次（每月變動） |
| **分布** | 二項分布（每位住院病人：死亡/未死亡） |
| **分母穩定性** | 變動（月住院人次隨季節、疫情波動） |
| **稀有程度** | 中等（區域醫院每月約數十例死亡） |
| **理論最佳** | **P Chart**（二項比例，分母可變） |
| **實務建議** | 若僅有已計算的千分率 → **I-MR Chart**；若可取得每月死亡數與住院人次 → **P Chart** |

**補充說明**：文獻指出死亡率是醫療 SPC 中最常被分析的變數之一。流行病學部門最常使用 P Chart 來監測此類指標。若需偵測小幅持續偏移，可額外搭配 **CUSUM** 圖。

#### 指標：非計畫性重返急診率（72hr）

| 維度 | 分析 |
|------|------|
| **數據本質** | 比率型（重返人次 / 急診離院人次 × 100） |
| **分子** | 72hr 內非計畫重返急診人次（計數） |
| **分母** | 急診離院人次（每月變動） |
| **分布** | 二項分布（每位離院病人：重返/未重返） |
| **分母穩定性** | 變動（急診量受季節、流感、假日影響大） |
| **稀有程度** | 中等（3-5% 的比率，分母夠大時事件數充足） |
| **理論最佳** | **P Chart** |
| **實務建議** | 若僅有百分比 → **I-MR Chart**；若有分子分母 → **P Chart** |

---

#### 指標：非計畫性重返加護病房（48hr）

| 維度 | 分析 |
|------|------|
| **數據本質** | 比率型（重返人次 / ICU 轉出人次 × 100） |
| **分子** | 48hr 內非計畫轉入 ICU 人次 |
| **分母** | 加護病房轉出人次（每月變動） |
| **分布** | 二項分布 |
| **分母穩定性** | 變動（ICU 轉出量波動較大） |
| **稀有程度** | **較稀有**（約 2%，且 ICU 轉出人數不多，p̄×n 可能 < 5） |
| **理論最佳** | **P Chart**，但需注意小樣本問題 |
| **實務建議** | **I-MR Chart**（因分母小、事件稀少，P Chart 管制界限不穩定） |

**專家備註**：ICU 轉出人數每月可能僅數十人，重返 ICU 每月可能 0-2 件。此時 p̄×n < 5，P Chart 的常態近似失效。建議使用 I-MR Chart，或採用精確二項界限的修正 P Chart。

### 2.2 加護病房照護

#### 指標：加護病房院內感染密度

| 維度 | 分析 |
|------|------|
| **數據本質** | 密度型（感染人次 / 住院人日 × 1000） |
| **分子** | ICU 感染人次（計數，可多次感染不同部位） |
| **分母** | ICU 住院人日（每月變動） |
| **分布** | **Poisson 分布**（事件為稀有計數，暴露量為人日） |
| **分母穩定性** | 變動（ICU 住院人日逐月不同） |
| **稀有程度** | 稀有（感染事件相對於人日機會極為稀少） |
| **理論最佳** | **U Chart**（Poisson 計數率，分母可變） |
| **實務建議** | 若有分子（感染件數）+ 分母（人日）→ **U Chart**；若僅有密度值 → **I-MR Chart** |

**專家備註**：感染密度是 U Chart 的經典應用場景。文獻中，Sellick 和 Morton 等人在流行病學部門即使用 U Chart 來監測院內感染。但若 Poisson 分布假設不成立（例如感染事件之間不獨立），Dr. Wheeler 建議退回使用 I-MR Chart。

### 2.6 感染管制

#### 指標：組合式照護執行率（CVC Bundle Compliance）

| 維度 | 分析 |
|------|------|
| **數據本質** | 比率型（完整執行次數 / 應執行次數 × 100） |
| **分子** | 組合式措施完整執行次數 |
| **分母** | 應執行次數（每月變動，依導管使用量） |
| **分布** | **二項分布**（每次機會：完整執行/未完整執行） |
| **稀有程度** | **反向稀有**（比率約 90%+，接近 100%） |
| **理論最佳** | **P Chart** |
| **實務建議** | **P Chart**（但需注意高比率時管制界限可能突破 100%，需截斷處理） |

**專家備註**：當 p̄ 接近 100% 時，UCL 可能超過 100%，此時 UCL 應截斷為 100%。同時 LCL 的偵測能力才是重點——落入 LCL 以下代表執行率異常下降。

### 2.8 關鍵照護

#### 指標：跌倒發生密度

| 維度 | 分析 |
|------|------|
| **數據本質** | 密度型（跌倒事件數 / 住院人日 × 1000） |
| **分子** | 跌倒事件數（同一病人可多次跌倒） |
| **分母** | 住院人日（每月變動） |
| **分布** | Poisson 分布 |
| **稀有程度** | 稀有（千分之 0.6） |
| **理論最佳** | **U Chart** |
| **實務建議** | **U Chart**（跌倒是 U Chart 在醫療品質中的經典應用場景） |

**專家備註**：SPC for Excel 文獻以「每 1000 病人日跌倒次數」作為 U Chart 的醫療範例。跌倒完全符合 Poisson 假設——事件為離散計數、發生在明確的時間空間（住院期間）、各事件獨立、事件相對機會為稀有。

---

#### 指標：約束盛行密度

| 維度 | 分析 |
|------|------|
| **數據本質** | 密度型（約束人日 / 住院人日 × 1000） |
| **分子** | 約束人日（**非整數事件計數**，而是天數加總） |
| **分母** | 住院人日 |
| **分布** | **非典型**（分子是人日而非事件計數，不符合 Poisson） |
| **稀有程度** | 中等 |
| **理論最佳** | **I-MR Chart** |
| **實務建議** | **I-MR Chart**（分子為連續型的天數加總，非離散事件計數，不適合 U Chart） |

**專家備註**：這是一個需要特別注意的指標。約束盛行密度的「分子」是約束人日（天數），而非事件次數。一個病人可能連續被約束 7 天，這 7 天是連續的、非獨立的。這違反了 Poisson 分布的「事件獨立」假設，因此 U Chart 不適用。I-MR Chart 作為最穩健的選擇是正確的。

### 2.9 呼吸照護（僅竹東）

#### 指標：呼吸器脫離率

| 維度 | 分析 |
|------|------|
| **數據本質** | 比率型（成功脫離人次 / 使用呼吸器人次 × 100） |
| **分子** | 成功脫離呼吸器人次 |
| **分母** | 使用呼吸器人次（每月變動，且數量不多） |
| **分布** | 二項分布（每位使用者：成功脫離/未脫離） |
| **稀有程度** | **反向稀有**（約 75%，但分母小） |
| **理論最佳** | **P Chart** |
| **實務建議** | **I-MR Chart**（竹東為小型院區，月呼吸器使用人次有限，分母太小） |

---

### 2.10 經營管理

#### 指標：病床使用率

| 維度 | 分析 |
|------|------|
| **數據本質** | **連續型比率**（佔床人日 / 可用病床人日 × 100） |
| **分子** | 佔床人日 |
| **分母** | 可用病床人日（相對穩定，除非病床數調整） |
| **分布** | **近似常態**（非二項、非 Poisson，是兩個連續量的比值） |
| **稀有程度** | 不適用（值域約 60-90%，無稀有問題） |
| **理論最佳** | **I-MR Chart** |
| **實務建議** | **I-MR Chart** |

**專家備註**：病床使用率是真正適合 I-MR Chart 的指標。它是一個連續型比率，每月一個數據點，且值域寬廣、不存在稀有事件問題。分母（可用病床人日）相對穩定，不像其他指標的分母會劇烈波動。

### 3.3 建議的分層策略

結合理論正確性與實務可行性，建議採用**雙層管制圖策略**：

```
Layer 1（基礎層）：所有指標一律繪製 I-MR Chart
  → 確保每個指標都有管制圖
  → 利用 I-MR 的穩健性作為基線

Layer 2（進階層）：當可取得分子/分母時，額外繪製正確圖型
  → P Chart：住院死亡率、重返急診率、SSI、急診暫留率、遵從率等
  → U Chart：感染密度、跌倒密度、ADE 通報率等
  → 提供更精確的管制界限（會隨分母變動而調整）
```

## 4. 管制界限計算公式

### 4.1 I-MR Chart

```
中心線 CL = X̄（所有觀測值的平均）
移動全距 MR = |X(i) - X(i-1)|
MR̄ = 移動全距的平均值

UCL = X̄ + 2.66 × MR̄
LCL = X̄ - 2.66 × MR̄

（2.66 = 3/d₂，其中 d₂ = 1.128 for n=2）
```

### 4.2 P Chart

```
中心線 p̄ = 總不良數 / 總樣本數 = Σd(i) / Σn(i)

UCL(i) = p̄ + 3 × √(p̄(1-p̄)/n(i))
LCL(i) = p̄ - 3 × √(p̄(1-p̄)/n(i))

注意：管制界限隨每月樣本數 n(i) 變動
      LCL 不得小於 0
      UCL 不得大於 1（若繪製比例）
```

### 4.3 U Chart

```
中心線 ū = 總事件數 / 總暴露量 = Σc(i) / Σn(i)

UCL(i) = ū + 3 × √(ū/n(i))
LCL(i) = ū - 3 × √(ū/n(i))

注意：管制界限隨每月暴露量 n(i) 變動
      LCL 不得小於 0
```

## 5. 對系統設計的影響

### 5.1 資料結構擴充

要支援正確的管制圖選型，**每個指標的數據點需要額外儲存分子和分母**：

```typescript
interface DataPoint {
  indicator_id: string;
  campus: string;
  year: number;
  month: number;
  value: number;         // 最終比率值（原有）
  numerator?: number;    // 分子（新增）
  denominator?: number;  // 分母（新增）
}
```

### 5.2 指標定義擴充

```typescript
interface Indicator {
  // ...原有欄位...
  
  // 管制圖相關（新增）
  chart_type: 'I-MR' | 'P' | 'U' | 'C' | 'G' | 'EWMA' | 'CUSUM';
  chart_type_rationale: string;   // 選型理由
  data_nature: 'continuous' | 'binomial_rate' | 'poisson_rate' | 'count';
  is_rare_event: boolean;         // 是否為稀有事件
  min_pn_threshold: number;       // p̄×n 的閾值（預設 5）
  needs_numerator_denominator: boolean;  // 是否需要分子分母才能用最佳圖型
}
```

### 5.3 智慧圖型選擇邏輯

系統應根據指標定義和實際數據，自動判斷可用的管制圖類型：

```typescript
function determineChartType(indicator: Indicator, dataPoints: DataPoint[]): ChartType {
  // 如果有分子分母數據
  const hasRawData = dataPoints.every(dp => 
    dp.numerator !== undefined && dp.denominator !== undefined
  );
  
  if (!hasRawData) {
    return 'I-MR'; // 退回 I-MR
  }
  
  // 檢查數據本質
  if (indicator.data_nature === 'continuous') {
    return 'I-MR';
  }
  
  if (indicator.data_nature === 'binomial_rate') {
    // 檢查 p̄×n 是否足夠
    const avgP = sum(numerators) / sum(denominators);
    const avgN = mean(denominators);
    if (avgP * avgN >= 5) {
      return 'P';
    } else {
      return 'I-MR'; // 稀有事件退回
    }
  }
  
  if (indicator.data_nature === 'poisson_rate') {
    // 檢查月平均事件數
    const avgEvents = mean(numerators);
    if (avgEvents >= 1) {
      return 'U';
    } else {
      return indicator.is_rare_event ? 'G' : 'I-MR';
    }
  }
  
  return 'I-MR'; // 預設
}
```

### 5.4 Excel 解析擴充

為了取得分子和分母，Excel 解析時需要嘗試辨識：

- 有些工作表會同時列出「分子」、「分母」、「比率」三列
- 有些只有「比率」一列
- 解析器應自動偵測是否存在分子/分母欄位

## 6. 總結與建議

### 6.1 核心發現

1. **不是所有指標都應該用 I Chart。** 在 21 個指標中，理論上只有 6 個最適合 I-MR Chart（非計畫重返 ICU、VAP、CLABSI、約束盛行密度、呼吸器脫離率、病床使用率）。

2. **P Chart 適用的指標最多（8 個）**，因為大量品質指標本質上是二項比例（每個個體 pass/fail）。

3. **U Chart 適用 4 個指標**，都是「事件計數/暴露量」的密度型指標。

4. **但受限於 QIP 報表可能僅提供已計算比率**，實務上很多指標可能必須退回使用 I-MR Chart。

5. **稀有事件是最大的技術挑戰。** VAP、CLABSI、處方錯誤等極稀有事件，傳統管制圖都不太好用，需要考慮 G Chart 或 CUSUM。

### 6.2 對 Claude Code 的建議

建議開發時採取**漸進式策略**：

- **Phase 1**：所有指標先實作 I-MR Chart（確保基本功能運作）
- **Phase 2**：為理論上適合 P Chart / U Chart 的指標，增加進階圖型（當有分子分母數據時）
- **Phase 3**：為極稀有事件指標，考慮增加 G Chart 或 CUSUM

這樣既能快速交付可用的儀表板，又能逐步提升統計嚴謹性。
